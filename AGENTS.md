# AGENTS.md

## 基本ルール

- すべてのやり取りは日本語で行ってください。
- 作業内容の説明、確認、提案、コミットメッセージも日本語で行ってください。
- ファイルを変更する前に、変更対象と変更理由を簡潔に説明してください。
- 既存の機能を壊さないことを最優先してください。
- 不明点がある場合は、勝手に大きく進めずに確認してください。

## Git操作ルール

- mainブランチへpushする前に、必ず現在の状態をセーブポイントとして保存してください。
- セーブポイント用ブランチ名は `savepoint/YYYYMMDD-HHMMSS-before-main-push` の形式にしてください。
- セーブポイントブランチを作成したら、必ずリモートにもpushしてください。
- その後、mainブランチに戻ってからmainへpushしてください。
- push前には `git status` を確認し、変更内容を説明してください。
- 可能な場合は、テストまたは起動確認を行ってからpushしてください。

## 触ってはいけない範囲

- `.env` や秘密鍵、APIキーが含まれるファイルは編集しないでください。
- ユーザーが明示しない限り、認証情報、決済情報、環境変数ファイルには触らないでください。
- `backup/`、`archive/`、`do-not-edit/` などの退避用ディレクトリは編集しないでください。
- 作業対象外のディレクトリには変更を加えないでください。
