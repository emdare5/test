# AGENTS.md

## 基本方針

- すべての会話、説明、確認、提案は日本語で行ってください。
- コミットメッセージも日本語で作成してください。
- 初心者にも分かるように、専門用語を使う場合は簡単な説明を添えてください。
- 作業前に、何を変更する予定かを簡潔に説明してください。
- 作業後に、何を変更したか、どのファイルを触ったか、次に何を確認すべきかを説明してください。

## 開発方針

- 既存の機能を壊さないことを最優先してください。
- 変更範囲は必要最小限にしてください。
- 大きな変更をする前には、必ず確認してください。
- 依存関係を追加する場合は、理由を説明してから提案してください。
- セキュリティ上危険な実装は避けてください。

## Git運用

- mainブランチにpushする前に、必ずタイムスタンプ付きのsavepointブランチを作成してください。
- savepointブランチ名は `savepoint/YYYYMMDD-HHMMSS-before-main-push` の形式にしてください。
- push前には `git status` と `git diff` を確認してください。
- コミットは意味のある単位で行ってください。
- コミットメッセージは日本語で、変更内容が分かるように書いてください。

## 禁止事項

- `.env`、秘密鍵、APIキー、認証情報を含むファイルを編集しないでください。
- ユーザーが明示しない限り、削除系コマンドを実行しないでください。
- `rm -rf` のような危険なコマンドは、必ず確認してから実行してください。
- 作業対象外のディレクトリやファイルには触らないでください。
- 本番環境への変更やデプロイは、必ず確認してから行ってください。

## 返答スタイル

- できるだけ簡潔に、でも初心者が迷わないように説明してください。
- 作業内容は箇条書きで整理してください。
- エラーが出た場合は、原因の仮説、確認したこと、次に試すことを分けて説明してください。
